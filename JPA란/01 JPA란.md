# JPA가 뭔가
```
JPA
Java Persistence API 

JPA는 자바 진영에서 ORM기술 표준으로 사용되는 인터페이스 모음임

실제 구현된 것이 아니라 구현된 클래스오 매핑을 해주기 위해서 사용되는 프레임워크임

JPA를 구현한 대표적인 오픈소스는 Hibernate가 있음
```


# ORM이란?
```
ORM

Object - Relational Mapping

일반적으로 알고 있는 어플리케이션 class와 RDB(relational data base)의 테이블을 매핑한느 것임

기술적으로 어플리케이션 객체를 RDB 테이블에 자동으로 영속화(persist, save ..) 해주는 거임
```

### ORM 장단점
```
장점
1) SQL문이 아닌 method를 통해 DB를 조작할 수 있음. 개발자는 객체 모델을 이용해 비즈니스 로직을구성하는데만 집중할 수 있음
(내부적으로 쿼리를 생성해서 DB를 조작함. 근데 개발자는 신경안써도 되는 거임)

2) 쿼리와 같이 필요한 선언문, 할당 등 부수적인 코드가 줄어듬. 각종 객체에 대한 코드를 별도로 작성해서 코드 가독성을 높임

3) 객체지향적인 코드 작성이 가능함. 오직 객체지향적인 접근만 고려하면 되기때문에 생산성이 증가함

4) 매핑 정보가 class로 명시되서, ERD를 보는 의존도를 낮출 수 있고, 유지보수 및 리팩토리에 유리함

5) 기존 방식에서 mySQL DB를 사용하다가 postgreSQL로 변환한다고 가정하면, 새로 쿼리를 짜야하는 경우가 생김. 이런 경우 ORM을 사용하면 쿼리를 수정할 필요가 없음.


단점
1) 프로젝트 규모 커지고 복잡해서 설계가 잘못된 경우, 속도 저하, 일관성 무너지는 문제 발생

2) 복잡하고 무거운 쿼리는 속도를 위해 별도 튜닝이 필요함. 결국 SQL을 써야함

3) 학습 비용이 비쌈
```


# JPA란
```
1) Java Persistance API

2) java 진여에서 ORM 기술 표준으로 사용하는 인터페이스 모음

3) 자바 어플리케이션에서 관계형 데이터베이스를 사용하는 방식을 정의한 인터페이스임

4) 인터페이스라서 Hibernate, Open JPA 등이 JPA를 구현함
```

## JAP 왜 쓸까
```
JPA는 반복적인 CRUD SQL를 처리해줌. JPA는 매핑된 관계를 이용해 SQL를 생성하고 실행하는데, 개발자는 어떤 SQL이 실행될지 생각만하면 되고 예측하기 쉬움

추가적으러 JPA는 네이티브 SQL기능 제공함
그래서 관계 매핑이 어렵거나 성능 이슈가 우려되면 SQL을 직접 작성해서 사용함

JPA를 사용 이점

SQL이 아닌 객체 중심으로 개발 할 수 있음
그래서 당연히 생산성이 좋고
유지보수도 수월함

JPA는 패러다임의 불일치도 해결함
예를 들어 JAVA에서 부모클래스, 자식클래스 가 잇는데
DB에서는 이러한 객체 상속관계를 지원하지 않음
JPA는 이를 아래 그림처럼 해결함
```
